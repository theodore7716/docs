# 任务条件

任务中心的每个任务都有一个或多个"完成条件"。系统通过自动检测用户在 App 内的操作来判断条件是否达成。不同任务的条件类型、目标值和统计口径各不相同。

## 条件如何运作

- 系统自动识别用户在 App 内的操作行为，无需手动提交或申请
- 条件达成后系统自动更新任务进度
- 部分条件有金额、次数、品种等具体要求，需仔细阅读任务详情
- 进度更新可能有短暂延迟，通常几分钟内完成同步
- 每日/每周/每月任务在周期结束后进度清零，重新开始计算

## 常见条件类型

| 条件 | 说明 | 统计口径 |
|------|------|---------|
| 成交股票 N 次 | 成功成交的股票交易次数 | 按成交笔数计算，挂单未成交不计入 |
| 股票交易金额达 N 美元 | 股票交易累计金额 | 按美元计价，多币种按汇率折算 |
| 基金申购金额达 N 美元 | 基金申购累计金额 | 按美元计价 |
| 打新 N 次 | IPO 认购次数 | 提交认购即计入 |
| 期权交易 N 次 | 期权成交次数 | 按成交笔数计算 |
| 期权交易 N 张 | 期权成交合约张数 | 按合约张数计算 |
| 盘前期权交易 N 张 | 盘前时段期权成交张数 | 仅统计盘前时段的成交 |
| 涡轮牛熊证交易 N 次 | 涡轮牛熊证成交次数 | 按成交笔数计算 |
| 涡轮牛熊证交易金额达 N 美元 | 涡轮牛熊证交易累计金额 | 按美元计价 |
| 碎股交易 N 次 | 碎股交易成功次数 | 按成交笔数计算 |
| 定投交易 N 次 | 定投成功次数 | 按成交笔数计算 |
| 使用网格交易 | 网格策略交易成功 | 完成一笔网格交易即可 |
| 使用卖空交易 | 卖空交易成功 | 完成一笔卖空交易即可 |
| 使用夜盘交易 | 夜盘时段交易成功 | 仅统计夜盘时段的成交 |
| 使用桌面端交易 | 桌面客户端完成交易 | 在桌面端发起并成交 |

所有交易类条件以"成交"为准，挂单未成交不计入进度。

### 入金与资产类

| 条件 | 说明 | 统计口径 |
|------|------|---------|
| 单笔入金 N 美元 | 单次入金金额达到指定阈值 | 以美元计价，其他币种按汇率折算 |
| 入金后资产达 N 美元 | 入金后账户总资产达到指定金额 | 入金到账后系统自动检测总资产 |
| 月日均资产达 N 美元/港币 | 当月日均资产达到指定阈值 | 按当月每日资产的平均值计算 |

资产类条件可能受市场波动影响，系统以实际计算结果为准。

## 任务条件认定答疑

**为什么我完成了操作但任务进度没更新？**

可能原因包括：
1. 系统延迟，进度更新有短暂延迟，通常几分钟内完成，请稍等后刷新页面
2. 未达到具体要求，任务对操作有具体限制（如金额、次数、品种），请仔细核对任务详情
3. 跨零点操作，跨零点的操作可能被计入下一个统计周期
4. 挂单未成交，交易类条件以"成交"为准，挂单未成交不计入

如刷新后仍未更新，请联系客服并说明账号和具体操作情况。

**系统怎么判断我是否完成了条件？**

系统自动检测用户在 App 内的操作行为，包括交易记录、开户状态、社区互动等。无需用户手动提交或申请，条件达成后系统自动更新进度并发放奖励。

**挂单但没成交算不算完成交易条件？**

不算。所有交易类条件均以"成交"为准，只有订单实际成交后才计入进度。挂单中、已撤单、部分成交（仅计入已成交部分）等状态不算完成。

**不同市场的交易次数是分开计算还是合并的？**

取决于具体任务的配置。部分任务指定了特定市场（如"完成香港股票交易"），则只统计该市场的交易；部分任务不限市场，则合并计算所有市场的交易。请以任务详情页的说明为准。

**我在其他平台做的交易算不算？**

不算。任务条件仅统计在本 App 内完成的操作，其他券商或平台的交易不会被系统检测到，也不会计入任务进度。

**任务进度多久更新一次？**

系统在检测到用户操作后会尽快更新进度，通常在几分钟内完成。部分条件（如月日均资产）可能按天更新。如果长时间未更新，请尝试刷新页面或联系客服。

**每日/每周/每月任务重置后进度会清零吗？**

会。每日任务在每天零点重置，每周任务在每周一零点重置，每月任务在每月 1 日零点重置。重置后上一周期的进度不保留，需要重新完成。

**同一个操作能推进多个任务的进度吗？**

能。例如完成一笔股票交易，可能同时满足每日任务"交易 1 次"、每周任务"交易 5 次"和成长任务"累计交易 50 次"的条件，系统会分别自动更新各任务的进度。

**任务包内的条件和普通任务的条件一样吗？**

条件类型可能相同，但进度独立计算。即使普通任务中的同类条件已完成，也不会自动计入任务包的进度。任务包内的每个条件需要单独完成。

**条件中的金额是按什么币种计算的？**

金额类条件通常以美元（USD）计价。如果用户交易使用的是其他币种（如港币、新加坡元等），系统会按汇率自动折算为美元。部分条件（如月日均资产）可能同时支持美元和港币，以任务详情页的标注为准。

**为什么任务显示需要"交易 5 次"但我交易了还不够？**

请注意以下几点：统计口径方面，"交易次数"按成交笔数计算，以系统实际统计为准；时间范围方面，每日/每周/每月任务只统计当前周期内的交易；品种限制方面，部分任务限定了具体的交易品种（如仅限股票、仅限期权等）；另外挂单未成交的订单不计入。请仔细阅读任务详情中的具体要求。

**连续签到/连续猜涨跌中断了怎么办？**

连续类条件要求每天不间断完成操作。一旦中断，连续天数会从头开始计算。目前不支持补签或恢复中断前的记录。建议每天定时完成操作以保持连续记录。

## 任务条件通用限制

- 进度更新通常在几分钟内完成，如未及时显示请刷新后查看
- 零点前后的操作可能被计入不同的统计周期，建议在截止时间前至少 30 分钟完成操作
- 所有交易类条件以"成交"状态为准，挂单中、已撤单不计入
- 金额类条件通常以美元计价，其他币种按汇率折算
- 每日/每周/每月任务在周期结束后进度清零，不结转至下一周期
- 任务包内的条件进度与普通任务相互独立，需分别完成
- 所有条件仅统计在本 App 内的操作，其他平台的操作不计入
- 连续签到、连续猜涨跌等条件一旦中断即从头计算，无法补回
- 所有时间判断以系统服务器时间为准，非用户设备本地时间
