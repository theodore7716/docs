# DDA 授权失败怎么办

DDA 分为两种不同服务，处理方式不同：

- **eDDA**（香港）：基于 FPS 转数快系统，用于港元入金
- **DDA**（新加坡）：直接扣款授权服务，用于新加坡元入金

## eDDA 授权失败（香港）

授权时提示错误，根据错误信息处理：

| 错误信息 | 原因 | 处理方式 |
|---------|------|---------|
| Debtor account name / ID not matched | 姓名或证件格式不符 | 确认填写大写英文姓名，证件类型与银行开户时一致 |
| Invalid debtor account number | 账号有误或使用了分币种账号 | 填写综合账号主账号，去掉银行代码前缀；部分银行港币账号末尾为 1 或 2 |
| Mandate identification already exists | 同时在银行端和 App 端发起了授权 | 等待其中一个完成或失效后，仅在 App 端重新发起 |
| Debtor account not found | 账号错误 | 核对银行账号，确认没有多余空格或错位 |
| Rejected by Other bank | 账户类型问题 | 确认绑定的是港元账户，而非美元账户 |
| Invalid Payer ID | 证件信息不一致 | 确认长桥账户证件信息与银行端一致 |

**华侨银行**：授权一般需 2 个工作日处理。若长时间无结果，检查是否选错了银行代码，可删除后重新发起。

**重置授权**：如排查后仍无法授权，联系银行端重置 eDDA，再在 App 内重新发起。

## DDA 授权失败（新加坡）

根据错误码处理：

| 错误码 / 提示 | 原因 | 处理方式 |
|-------------|------|---------|
| 1209（Refer to your payer/payee） | 银行账户余额或额度不足 | 确保银行账户余额充足后重新授权 |
| 1100（Other reason） | 曾在银行端删除过 DDA 绑定 | 联系长桥客服关闭旧记录，再重新授权 |
| ID number verification failed | 长桥注册的身份证号与银行端不一致 | 确认填写的证件号与银行开户证件一致 |
| BA3002（Consent not given） | 未完整阅读条款就点击了取消 | 重新发起授权流程，完整阅读条款后提交 |

## 授权成功但无法发起入金

授权状态正常但点击 DDA/eDDA 入金时报错：

1. 检查银行账户余额是否充足（账户余额不足时部分银行会拒绝扣款）
2. 确认银行卡未被冻结或限制
3. 检查长桥 App 内显示的授权状态是否为「已授权」

## 仍无法解决

联系客服时提供：

- 错误截图（含完整错误信息）
- 使用的银行名称
- 授权失败的时间

**在线客服**：长桥 App → 我的 → 帮助与客服 → 在线客服
